Overview

What is Temporal?

namespace

Temporal is a global namespace (like Math or Intl) — not a constructor. It is the modern replacement for Date, with proper time-zone handling, calendars, immutability, nanosecond precision, and a clear separation between absolute instants, wall-clock times, and durations. 

Temporal
├── .Now                  // current time helpers
├── .Instant             // fixed point on the timeline (ns since epoch)
├── .ZonedDateTime       // instant + time zone + calendar
├── .PlainDateTime       // wall-clock date+time (no zone)
│   ├── .PlainDate       // year + month + day
│   │   ├── .PlainYearMonth
│   │   └── .PlainMonthDay
│   └── .PlainTime       // hour..nanosecond
└── .Duration            // difference between two time points
Rule of thumb: use Instant for timestamps, ZonedDateTime for "real-world" events, and Plain* when the time zone is irrelevant (birthdays, business hours, recurring alarms).

Key principles

concepts
Immutable All methods return new instances
Precise Nanosecond resolution via BigInt
Explicit Time zones & calendars are first-class
Type-safe Separate types prevent misuse
ISO default ISO 8601 calendar unless specified

Temporal.Now

Getting the current time

Temporal.Now 
Temporal.Now.instant()
  // → Temporal.Instant (nanosecond timestamp)

Temporal.Now.zonedDateTimeISO()
  // → ZonedDateTime in system zone, ISO calendar

Temporal.Now.zonedDateTimeISO("Asia/Kolkata")
  // → ZonedDateTime in a specific zone

Temporal.Now.plainDateTimeISO()
Temporal.Now.plainDateISO()
Temporal.Now.plainTimeISO()

Temporal.Now.timeZoneId()
  // → "Asia/Kolkata" (system zone string)

Epoch readings

number / bigint 
const now = Temporal.Now.instant();

now.epochMilliseconds;   // number
now.epochNanoseconds;    // bigint

// Equivalent of Date.now():
Temporal.Now.instant().epochMilliseconds;

Temporal.Instant

Creating instants

point in time

A single point on the global timeline. No zone, no calendar.

Temporal.Instant.from("2025-12-25T10:00Z");
Temporal.Instant.from("2025-12-25T10:00+05:30");

Temporal.Instant.fromEpochMilliseconds(1735128000000);
Temporal.Instant.fromEpochNanoseconds(1735128000000000000n);

// From legacy Date:
Temporal.Instant.fromEpochMilliseconds(new Date().getTime());

Arithmetic & comparison

methods 
const a = Temporal.Instant.from("2025-01-01T00:00Z");
const b = a.add({ hours: 3, minutes: 30 });

a.since(b);                    // Duration (b → a)
a.until(b);                    // Duration (a → b)
a.equals(b);                   // boolean
Temporal.Instant.compare(a, b); // -1 | 0 | 1

a.round({ smallestUnit: "minute" });

Pin to a zone / calendar

conversion 
const inst = Temporal.Now.instant();

inst.toZonedDateTimeISO("America/New_York");
  // → ZonedDateTime

inst.toString();                 // ISO 8601 (UTC)
inst.toLocaleString("en-US");
inst.toJSON();

Temporal.ZonedDateTime

The "complete" type — instant + time zone + calendar

zoned date-time

Use this for any real-world moment that needs a calendar & time-zone interpretation. Handles DST and offset transitions correctly. 

// From an RFC 9557 string:
Temporal.ZonedDateTime.from("2025-12-25T09:00[Asia/Kolkata]");

// From a property bag:
Temporal.ZonedDateTime.from({
  year: 2025, month: 12, day: 25,
  hour: 9, minute: 0,
  timeZone: "Asia/Kolkata",
  calendar: "iso8601"                // default
});

// Disambiguation for DST gaps/overlaps:
Temporal.ZonedDateTime.from(input, {
  disambiguation: "compatible" // | "earlier" | "later" | "reject"
});

Field access

getters 
const z = Temporal.Now.zonedDateTimeISO();

z.year; z.month; z.day;
z.hour; z.minute; z.second;
z.millisecond; z.microsecond; z.nanosecond;

z.dayOfWeek;     // 1 (Mon) … 7 (Sun)
z.dayOfYear;     // 1 … 366
z.weekOfYear;
z.yearOfWeek;
z.daysInMonth;
z.daysInYear;
z.inLeapYear;

z.timeZoneId;    // "Asia/Kolkata"
z.offset;        // "+05:30"
z.offsetNanoseconds;
z.hoursInDay;    // usually 24, 23/25 on DST
z.epochMilliseconds;
z.epochNanoseconds;

Modify fields (immutably)

with / arithmetic 
z.with({ hour: 9, minute: 0 });
z.withCalendar("japanese");
z.withTimeZone("Europe/London");
z.withPlainTime("14:30");

z.add({ days: 7 });
z.subtract({ hours: 2, minutes: 30 });

z.round({ smallestUnit: "hour" });
z.startOfDay();              // midnight in this zone

z.getTimeZoneTransition({ direction: "next" });
  // → next DST change as ZonedDateTime | null

Temporal.PlainDateTime

Wall-clock date + time

no time zone

Use when the time zone is unknown, irrelevant, or set later (forms, schedules).

Temporal.PlainDateTime.from("2025-12-25T09:30:00");

Temporal.PlainDateTime.from({
  year: 2025, month: 12, day: 25,
  hour: 9, minute: 30
});

// Overflow control:
Temporal.PlainDateTime.from(
  { year: 2025, month: 2, day: 30 },
  { overflow: "reject" }   // throws (default: "constrain")
);

Common methods

API 
const dt = Temporal.PlainDateTime.from("2025-12-25T09:30");

dt.with({ hour: 12 });
dt.withCalendar("hebrew");
dt.withPlainTime("00:00");

dt.add({ months: 1 });
dt.subtract({ days: 10 });
dt.since(other);
dt.until(other);

dt.round({ smallestUnit: "minute" });

dt.toZonedDateTime("Asia/Kolkata");
dt.toPlainDate();
dt.toPlainTime();

Temporal.PlainDate

Calendar dates

year-month-day

Ideal for birthdays, anniversaries, deadlines — anything where time-of-day is irrelevant.

Temporal.PlainDate.from("2025-12-25");
Temporal.PlainDate.from({
  year: 2025, month: 12, day: 25
});

// Non-ISO calendar:
Temporal.PlainDate.from("2025-12-25[u-ca=hebrew]");

Arithmetic & conversion

examples 
const d = Temporal.PlainDate.from("2025-12-25");

d.dayOfWeek;            // 4 (Thu)
d.daysInMonth;          // 31
d.inLeapYear;           // false

d.add({ days: 30 });
d.subtract({ months: 1 });

// Difference as Duration:
const diff = d.until(
  Temporal.PlainDate.from("2026-01-01"),
  { largestUnit: "day" }
);
diff.days;  // 7

d.toPlainDateTime("09:00");
d.toZonedDateTime({
  timeZone: "Asia/Kolkata",
  plainTime: "09:00"
});
d.toPlainYearMonth();
d.toPlainMonthDay();

Temporal.PlainTime

Wall-clock time, no date

HH:MM:SS.sssssssss

For recurring daily times: an alarm at 8:00, lunch break at 12:30, etc.

Temporal.PlainTime.from("09:30:00");
Temporal.PlainTime.from({ hour: 9, minute: 30 });

const t = Temporal.PlainTime.from("09:30");

t.hour; t.minute; t.second;
t.millisecond; t.microsecond; t.nanosecond;

t.with({ minute: 45 });
t.add({ hours: 2 });   // wraps at 24h, no overflow
t.round({ smallestUnit: "minute" });

Temporal.PlainTime.compare(t1, t2);
t.equals(t2);

Temporal.PlainYearMonth & Temporal.PlainMonthDay

PlainYearMonth

year + month

Billing cycles, magazine issues, monthly stats.

const ym = Temporal.PlainYearMonth.from("2025-12");

ym.year;             // 2025
ym.month;            // 12
ym.daysInMonth;      // 31
ym.monthsInYear;     // 12

ym.add({ months: 1 });
ym.until(other, { largestUnit: "month" });

ym.toPlainDate({ day: 15 });

PlainMonthDay

month + day

Annually recurring events: birthdays, holidays.

const md = Temporal.PlainMonthDay.from("12-25");

md.monthCode;        // "M12"
md.day;              // 25

// Project onto a specific year:
md.toPlainDate({ year: 2026 });
  // → PlainDate 2026-12-25

md.equals(other);
md.with({ day: 31 });

Temporal.Duration

Creating durations

time span 
Temporal.Duration.from({
  hours: 2, minutes: 30
});

// ISO 8601 duration string:
Temporal.Duration.from("PT2H30M");     // 2h30m
Temporal.Duration.from("P1Y2M10DT2H30M");
Temporal.Duration.from("-PT5M");        // negative

// Common: result of since()/until() on any class
const d = a.until(b);

Fields & sign

properties 
const d = Temporal.Duration.from("P1Y2M3DT4H5M6S");

d.years; d.months; d.weeks;
d.days; d.hours; d.minutes;
d.seconds; d.milliseconds;
d.microseconds; d.nanoseconds;

d.sign;    // -1 | 0 | 1
d.blank;   // true if all-zero

Arithmetic, round, total

methods 
d.with({ hours: 10 });
d.negated();
d.abs();
d.add(other);
d.subtract(other);

// Normalize / round:
d.round({
  largestUnit: "hour",
  smallestUnit: "minute",
  relativeTo: zdt          // required for calendar units
});

// Total as a single unit:
d.total({ unit: "second" });
  // → number

Temporal.Duration.compare(d1, d2, { relativeTo });

Shared interface across classes

Most Temporal classes share the same method shape

reference table
Category Methods Notes
Construct new Class(), Class.from() .from() accepts strings or property bags
Update .with(), .withCalendar(), .withTimeZone(), .withPlainTime() All return new instances
Arithmetic .add(), .subtract() Accepts Duration or property bag
Diff .since(other), .until(other) Both return Duration
Round .round({ smallestUnit, roundingMode }) Not on PlainDate / PlainYearMonth / PlainMonthDay
Compare .equals(), Class.compare(a, b) compare returns -1, 0, 1
Serialize .toString(), .toJSON(), .toLocaleString() JSON-safe RFC 9557 strings
Value .valueOf() Throws — prevents accidental coercion with +/-/</>
No valueOf: Temporal objects intentionally throw on numeric coercion. Always use Class.compare() or .equals() — never <, >, or ==.

Calendars

Working with non-ISO calendars

calendarId

Default is "iso8601". Pass another id to interpret fields in that system.

const hebrew = Temporal.PlainDate.from({
  year: 5786, month: 3, day: 14,
  calendar: "hebrew"
});

hebrew.calendarId;       // "hebrew"
hebrew.era;              // "am"
hebrew.eraYear;          // 5786
hebrew.monthCode;        // "M03"

// List supported calendars:
Intl.supportedValuesOf("calendar");

// Convert calendar of an existing date:
hebrew.withCalendar("iso8601");

Calendar gotchas

tips
  • Use era + eraYear together; or just year. Don't mix.
  • Don't assume 12 months — use monthsInYear.
  • Don't assume 365/366 days — use daysInYear.
  • A leap year may add a leap month, not a leap day.
  • For month names, format with toLocaleString — don't hard-code.
  • Use monthCode for storage (e.g. birthdays); month for ordering.

RFC 9557 string format

The serialization grammar

spec

All Temporal types serialize via toString()/toJSON() using RFC 9557 (extension of ISO 8601 / RFC 3339).

// Full form:
YYYY-MM-DD T HH:mm:ss.sssssssss Z|±HH:mm [tz_id] [u-ca=cal_id]

// Real examples:
"2025-12-25"                                         // PlainDate
"2025-12"                                            // PlainYearMonth
"--12-25"                                            // PlainMonthDay
"09:30:00"                                          // PlainTime
"2025-12-25T09:30:00"                               // PlainDateTime
"2025-12-25T09:30:00Z"                              // Instant (UTC)
"2025-12-25T09:30:00+05:30"                         // Instant (offset)
"2025-12-25T09:30:00+05:30[Asia/Kolkata]"           // ZonedDateTime
"2025-12-25T09:30:00+05:30[Asia/Kolkata][u-ca=hebrew]"

// Duration (ISO 8601):
"P1Y2M10DT2H30M"     // 1 year, 2 months, 10 days, 2h30m
"PT1.5H"             // 1.5 hours
"-PT5M"              // negative 5 minutes

Common gotchas

Pick the right type

when to use what
Logging / timestamps Instant
Meeting at 3pm NYC ZonedDateTime
Birthday card PlainMonthDay
Daily alarm 8:00 PlainTime
Invoice month PlainYearMonth
Deadline date PlainDate
Local form input PlainDateTime
Elapsed time Duration

Top mistakes

avoid
  • Using < or > on Temporal objects — they throw. Use Class.compare().
  • Treating Instant as a date — it has no calendar fields. Convert to ZonedDateTime first.
  • Skipping relativeTo when rounding/totaling durations with calendar units (years, months, weeks).
  • Assuming a PlainDateTime identifies an instant — it doesn't until you attach a time zone.
  • Forgetting DST: a day can be 23 or 25 hours; use hoursInDay.
  • Using year across calendars for comparison — calendars are not interchangeable.

Disambiguation & overflow

options 
// Disambiguation (DST gaps/overlaps):
"compatible"  // default - matches legacy Date
"earlier"     // pick the earlier offset
"later"       // pick the later offset
"reject"      // throw RangeError

// Overflow (e.g. Feb 30):
"constrain"   // default - clamp to valid (Feb 28/29)
"reject"      // throw RangeError

// Rounding modes (round / since / until / total):
"halfExpand"  // default
"ceil", "floor", "trunc",
"halfCeil", "halfFloor",
"halfTrunc", "halfEven"
